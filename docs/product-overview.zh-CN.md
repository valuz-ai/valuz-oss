## 1. Valuz OSS 是什么

### 一句话定位

**一个工作台统管你所有的 Agent —— 让它们在真实项目里协同干活，跑在你自己的机器上。**

Valuz OSS 是一个开源、**本地优先（local-first）的 Agent 工作站**。你组建一支智能体团队，
每个智能体跑在你选择的 Runtime 与模型上，把它们放进真实项目里干活——规划、派单、把任务推到完成。
Agent loop 和你的全部数据都留在你自己的机器上；唯一的对外流量是你配置的 LLM 服务。

### 三个让它与众不同的点

这是核心亮点——按"对手能否复制"的难度排序：

1. **本地优先、完全自控。** Agent loop、你的文件、你的数据都跑在你的机器（或你自己的局域网/服务器）上，
   没有任何"只能托管"的部分。大厂的 Agent 产品在结构上就是 hosted-only——让你自控等于拆掉它的商业模式。
2. **Runtime 与模型中立。** 不绑定任何厂商。每个智能体跑在你选的 Runtime 上——
   **Claude Agent**、**Codex Agent** 或 **Valuz Agent**——配上你自己的模型通道。自由组合。
3. **天生多智能体：Project-as-Agent-Team + 目标驱动的 Task。** 项目不是某一个 Agent 的聊天窗，
   而是一支智能体团队的容器，他们作为职能流水线协作。一个 **lead（牵头）** 智能体规划工作、
   把子任务**派发**给 **member（成员）** 智能体、审阅产出、并把整件事推向目标。

### 与同类产品的差异

| 别的产品 | 它们的范式 | Valuz OSS |
|---------|-----------|-------------|
| Claude Code · Codex · opencode · Cursor | 单个 Agent 在一个 session 里干活 | 一支智能体**团队**在项目里协作 |
| IM 式的 Agent 群（群聊里 @ 多个 Agent） | 互相独立的 Agent 在频道里聊天 | 项目内的职能流水线——工作以**任务**流转，而非消息 |
| 托管式自治 Agent（Devin、Manus、托管服务） | 只能托管、单条长任务 | **自托管**，数据与算力自控，项目持续演进 |

### 不绑定任何行业

开源产品是面向任意"项目化知识工作"的通用 Agent 工作站——研究、写作、策划、产品设计、运维等等。
领域深度以**行业版（vertical）**的方式叠加在上面（第一站是投资研究，连接 Reportify 解锁——见 §3.10）。
投资研究是 Valuz OSS 的一种应用，而非它的定义。

### 部署形态

- **桌面应用** — 面向 **macOS** 和 **Linux** 的原生客户端：常驻托盘、全局快捷键、本地文件拖拽。
- **本地 / 局域网 headless + WebUI** — 以无 GUI 方式运行工作站，从你自己网络里的浏览器访问。
- **单租户，数据不出门。** 你的工作空间是你的；除了你授权的 LLM 调用，没有任何东西离开你的机器。

### 支持的 Runtime

Valuz OSS 不自己训模型，也不锁定某一家厂商。它是调度器。每个智能体声明它运行其上的 Runtime：

| Runtime | 典型用途 |
|---------|----------|
| **Claude Agent** | 长程推理、深度研究 |
| **Codex Agent** | 代码任务、工程化场景 |
| **Valuz Agent** | 灵活、可投研定制的 Agent |

模型通道可以是**你自己的 API Key**（OpenAI / Anthropic / 兼容 API），
也可以是 **Claude / Codex 订阅的 OAuth 登录**。Valuz 从不代理你的 LLM 调用——
凭证存在你的系统钥匙串里。

## 2. 核心概念词典

理解这些名词是理解产品的前提。

| 概念 | 定义 |
|------|------|
| **智能体（Agent）** | 一等的数字员工。每个智能体有身份（名字、说明、头像）、工作方法（instructions / system prompt）、大脑（Runtime + 模型 + 推理 effort）、以及**装备**（自带的技能 + 连接器）。智能体存放在**智能体库**里。 |
| **智能体库（Agent Library）** | 定义和维护所有智能体的全局之处。没有"模板/实例"双层——复制一个智能体 = 一个新的智能体。 |
| **项目（Project）** | 一支**智能体团队**的容器，围绕一摊工作开展。承载派驻的智能体、项目上下文、知识库、文件树、定时任务。 |
| **派驻（进项目）** | 把一个智能体加入项目团队。它是**实时引用（live reference）**，不是复制——在智能体库里编辑该智能体，它派驻的每个项目立即跟上。 |
| **任务（Task）** | 目标驱动的多智能体工作单元。一个 **lead** 智能体把它规划成若干子任务，派发给 **member** 智能体，审阅结果，并完成它。 |
| **Lead / Member** | 任务中，**lead** 拥有计划并派单；**member** 是工作者，各自完成一个子任务并回报。 |
| **运行 / 会话（Run / Session）** | 一次执行——一段对话或一个任务。会话之间相互隔离，各自保留历史。 |
| **对话 / 新对话** | 开始交流的入口：选 project + 智能体（项目内对话）、只选智能体（临时对话）、或都不选（用默认模型的快速对话）。 |
| **Runtime** | 会话运行其上的 Agent 引擎——Claude Agent / Codex Agent / Valuz Agent。 |
| **模型通道（Provider）** | 智能体使用的 LLM 连接——你自己的 API Key，或 Claude / Codex 订阅登录。 |
| **技能（Skill）** | 封装好、可复用的能力（"怎么做某件事"）——智能体装备的一部分。 |
| **连接器（Connector）** | 智能体可调用的外部工具（内置投研工具或你自己的 MCP Server）——智能体装备的一部分。 |
| **知识库（Docs）** | 智能体推理时检索的私有文档。 |
| **定时任务 / 自动化** | 按计划运行的 Agent 指令。 |
| **活动（Activity）** | 正在运行什么的总览——一眼看清所有对话与任务 lead。 |
| **Reportify** | 可选的云端平台，解锁投资研究行业版：官方 Skill、投研数据工具、云端高级解析。 |

## 3. 功能全景（按模块）

### 3.1 智能体（Agents）

> **角色**：劳动力。智能体是一等公民，定义一次，按需派驻到任何地方。

智能体给每份工作都带上整套装备。你配置四个分组：

- **身份** — 名字、一句话说明、头像。
- **工作方法** — 定义它如何干活的 instructions / system prompt（带一个引导骨架：角色定位、工作方法、
  输出纪律、边界）。
- **大脑** — 它的 Runtime（Claude Agent / Codex Agent / Valuz Agent）、默认模型通道、推理 effort。
- **装备** — 它携带的技能与连接器。

智能体在**智能体库**里维护（卡片列表、详情/编辑、新建）。没有"模板/实例"层：智能体*本身*就是你维护的对象，
复制一份就是一个新智能体。智能体派驻进项目时带着自己的装备——项目不二次配置它。

### 3.2 项目 —— Agent 团队

> **角色**：工作场所。项目是一支**智能体团队**的容器，而非某一个 Agent 的工作目录。

你创建一个项目（默认专属目录，或绑定一个已有的本地目录），然后从库里把智能体**派驻**进去——
每个都是**实时引用**，所以在库里改进一个智能体，会立刻升级用到它的每个项目。

一个项目承载：

- **项目成员** — 派驻到本项目团队的智能体。
- **项目上下文** — 你定义的 *Instructions*（方向、框架、输出偏好），加上在后台自动积累关键事实与进展的 *Memory*。
- **知识库** — 团队检索的私有文档。
- **文件树** — 浏览、引用、读写绑定目录。
- **定时任务** — 自动化重复性工作。

随着 session 推进，团队越来越"懂"这摊工作。适合持续、深度的工作：研究、报告撰写、长期项目。

### 3.3 任务 —— 目标驱动的多智能体协作

> **角色**：一支团队如何把一个目标干完——一起干，并干到完成。

任务是多个智能体协作之处。不是一个智能体独自硬磕，而是一个 **lead** 智能体来编排：

1. **规划。** lead 把目标拆成一个结构化计划——带依赖关系的若干子任务（哪些可并行、哪些被谁阻塞）。
   完整计划（含尚未开始的步骤）以实时面板可见。
2. **派发。** lead 把每个就绪的子任务交给一个 **member** 智能体。member 作为兄弟并行运行，
   互不依赖的子任务同时推进。
3. **审阅。** member 完成后，lead 审阅它——**通过**（解锁依赖它的步骤）或**打回返工**并附反馈。
4. **完成。** 任务收尾为 completed 或 failed。

因为工作锚定在一个**目标**上，派出去的子任务会被推到真正完成——而不是一轮就半途而废。
工作以任务在计划中流转，而非以消息在聊天里流转。

### 3.4 对话与新对话

> **角色**：快速入口。几秒钟就能开聊。

"新对话"是一个入口，不是一个智能体：

- **选 project + 智能体** → 项目内对话（落到该项目的 sessions）。
- **只选智能体** → 与该智能体的临时对话，无项目绑定。
- **都不选** → 用默认模型的快速对话。

对话会流式展示消息、展示 Tool Call 卡片便于你跟随推理、接受文件上传（用内置解析读取）、
并把产出物保留在上下文面板里。适合快速提问和一次性分析。

### 3.5 技能与连接器（智能体装备）

技能与连接器是智能体的**装备**——它们跟着智能体进入每个项目。

**技能（Skills）** — 封装好、可复用的能力：

- **两种调用方式** — 智能体在相关时自动调用，或你用 `/技能名` 或 `@技能名` 直接触发。
- **两类** — **自定义 Skill**（本地编写，无需云端）与**官方 Skill**（随客户端分发，连接 Reportify 后解锁）。
- **添加 Skill** — 用 AI 创建（内置 skill-creator）、从 URL 导入（如 GitHub）、或上传文件夹。
  编辑器展示完整目录树，允许编辑保存其中每个文件。

**连接器（Connectors）** — 智能体可调用的外部工具：

- **自定义 MCP Server** — 任意时候添加你自己的 HTTP MCP Server，无需云端连接。
- **内置投研工具** — 行情、研报库等，连接 Reportify 后可用。它们作为透明的 Agent 工具出现在 Tool Call 日志中。

### 3.6 知识库

知识库（"Docs"）存放智能体推理所依据的私有文档。

- **导入** PDF / Word / Markdown / Excel / CSV。
- **自动检索**，并**来源可见**，你能看到用了哪些片段。
- **管理** 文档：查看、删除、重新索引。
- **默认本地，连接后升级云端。** 解析默认本地运行；连接 Reportify 后升级为精度更高、格式更多。

### 3.7 定时任务与自动化

让 Agent 指令自动运行——如每天早上抓取行业数据、每周五生成报告。运行按工作空间分组、保留执行日志、完成时通知。

### 3.8 Runtime 与模型通道

- **Runtime** 按智能体设置 — Claude Agent / Codex Agent / Valuz Agent。
- **模型通道** — 你自己的 API Key（OpenAI / Anthropic / 兼容）或 Claude / Codex 订阅登录。
  暴露模型列表的通道自动探测；订阅型通道自带推荐清单。
- **会话开始后锁定** — 会话的 Runtime、通道、模型在创建后固定；模型不可中途切换。

智能体调用的每个工具都在 Tool Call 日志中可见。

### 3.9 活动（Activity）

工作站内正在运行什么的统一总览——快速对话、项目对话、任务 lead——
让你随时知道有什么在进行，而无需逐个打开。

### 3.10 行业版与 Reportify 连接

开源产品是行业中立的。领域深度以**行业版（vertical）**叠加在上面——第一站是**投资研究**。
连接 **Reportify** 是可选的，解锁三大件：

1. **官方投研 Skill** — 专业打造的投研能力。
2. **投研数据工具** — 行情、研报库、宏观数据，作为 Agent 连接器暴露。
3. **云端高级文件解析** — 比本地解析精度更高、格式更多。

不连接时，产品依然完全可用——用你自己的 API Key、自定义 Skill 和本地解析。

## 4. 版本规划：Open Core

Valuz OSS 采用 **Open Core** 模式，分三层：

- **开源版（本仓库）** — 完整的单租户工作站：桌面端 + 本地/局域网 headless + WebUI。
  智能体、项目、任务、技能、连接器、知识库、定时任务。你的资源、你的机器、数据不出门。
- **商业版** — 叠加在开源核心之上的付费产品：SaaS 托管的公共资源（精选模型、Skill、连接器、模板）、
  可选云端同步、以及团队能力（组织、成员、统一采购、协作）。
- **行业版** — 通用产品加上行业 overlay（智能体、技能、任务模板、合规包）。第一站：金融投资研究。

开源版免费，覆盖完整的单用户工作站。

## 5. 桌面体验

- **系统托盘** — 最小化到托盘；右键菜单：显示主窗口 / 新对话 / 退出。
- **全局快捷键** — 在任意位置唤起或隐藏主窗口（`⌘ + Shift + R`）。
- **文件拖拽** — 把文件拖到输入框，让智能体分析它或导入知识库。
- **通知** — 长时间任务或定时任务完成时提醒。
- **窗口管理** — 关闭窗口最小化到托盘（继续运行）；`⌘ + Q` 真正退出。

## 6. 产品地图

左侧导航在整个工作站中一致：

- **上部** — 新对话入口，以及 Projects 列表。
- **中部** — Recents：所有 session 按时间混排。
- **下部** — 全局菜单：**智能体 · 知识库 · 技能 · 定时任务 · 设置**，以及看运行态的 **活动（Activity）**。

中心是对话 / 任务区（消息流、输入框、Tool Call 卡片，以及——任务场景下——实时计划面板）。
项目在右侧追加一个可关闭的 Context Panel（项目上下文、成员、文件树）。

| 区域 | 含义 |
|------|------|
| 新对话 | 开始一段对话（项目内、与某个智能体、或快速） |
| 项目（Projects） | 你的智能体团队工作空间 |
| 活动（Activity） | 一眼看清正在运行什么 |
| 智能体（Agents） | 智能体库 |
| 知识库 | 私有文档管理 |
| 技能（Skills） | 可复用的智能体能力 |
| 定时任务 | 定时任务管理 |
| 设置 | 账号、模型通道、连接器、外观、快捷键、解析 |
