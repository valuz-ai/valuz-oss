# Valuz 使用手册

这是 Valuz 的产品使用手册，供「Valuz 小助手」回答「这个产品怎么用」时参考。
内容以这里为准；手册没覆盖的细节，老实说「不确定」，别编造。

## 1. Valuz 是什么

Valuz 是一个 Agent（智能体）协作平台。核心心智模型是：

> **项目 = 一个 Agent 团队协作的工作区。**

你把一批各有专长的 Agent「派驻」进一个项目，它们就能在这个项目里分工协作、
推进任务。需要随手问点什么、不想建项目时，用「临时对话」就够了。

## 2. 核心概念

- **Agent（智能体）**：可配置的执行单元。每个 Agent 有：
  - **指令（instructions）**：它的人设和工作方法（system prompt）。
  - **模型通道**：运行时（runtime）+ 模型 + 推理强度（effort）。
  - **技能（skills）**：可插拔的能力模块（如写作、检索）。
  - **装备（connectors / MCP）**：外部工具连接器。
- **Agent 库**：你拥有的所有 Agent 的集合（左侧「智能体」入口）。
- **项目（project）**：一个工作区。把库里的 Agent **派驻** 进来即可参与该项目。
- **派驻（deploy）**：把库里的一个 Agent 引用进某个项目（是引用，不是复制；
  改这个 Agent 会影响所有派驻它的项目）。
- **临时对话**：不绑定项目的快速对话，适合随手问、轻量任务。
- **会话（session）**：一次具体的对话/任务执行记录。

## 3. 常见操作（给步骤 + 入口）

- **配置模型通道**：设置 → 模型。连接一个 Provider（Claude / OpenAI / API Key 等），
  选一个默认模型。没配模型，任何 Agent 都跑不起来。
- **创建 Agent**：「智能体」页右上角「+」→ 填名称、一句话定位、模型，
  写指令；技能/装备按需在「增强能力」里加。创建后可在详情页继续微调。
- **复制现有 Agent**：在「智能体」列表里，对目标 Agent hover →「···」→「复制」，
  它的配置会带进创建表单，改改即可。
- **创建项目**：「项目」页新建一个项目（绑定一个本地目录作为工作区）。
- **把 Agent 派驻进项目**：在 Agent 详情页点「派驻到项目」选一个项目；
  或在项目里「添加智能体」从库里选。
- **开始临时对话**：首页「新对话」，在输入框上方选一个 Agent，直接聊。
- **给 Agent 加技能/装备**：Agent 详情页 →「技能」/「装备」标签 →「添加」。
- **接 MCP 连接器**：设置 / 连接器里配置 MCP server，再在 Agent 的「装备」里挂上。

## 4. 最佳实践

- **临时对话 vs 项目**：随手问、一次性的小任务 → 临时对话；需要多角色协作、
  反复推进、留痕的工作 → 建项目并派驻一个团队。
- **遇到专业深度任务**（投研、写作、产研等）：别用通用助手硬扛，
  在项目里派一个更专业的 Agent，或从预设团队里选。
- **模型 / 推理强度**：日常问答用默认即可；需要深度推理的复杂任务调高 effort。
